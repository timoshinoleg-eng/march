# Чек-лист: 7 проверок перед запуском чат-бота

## Проверка 1: Цель бота чётко сформулирована
- [ ] Я знаю, какую конкретную задачу решает бот
- [ ] У меня есть метрика успеха (например, «сократить время ответа с 2 часов до 5 минут»)
- [ ] Я понимаю, чем бот НЕ будет заниматься (границы ответственности)

**Если не отмечено:** Не запускайте. Нет цели = нет результата.

---

## Проверка 2: Сценарии покрывают 80% запросов
- [ ] Я собрал реальные вопросы клиентов за последний месяц
- [ ] Топ-10 вопросов покрыты сценариями бота
- [ ] Есть план для редких запросов (перевод на оператора)

**Как собрать:** Выгрузите переписки из Telegram/WhatsApp за месяц. Сгруппируйте похожие вопросы. Первые 3-5 групп — ваши приоритеты.

---

## Проверка 3: Интеграции протестированы
- [ ] CRM получает данные из бота (проверено тестовой заявкой)
- [ ] Уведомления приходят нужным людям
- [ ] Если бот «застрял» — оператор узнаёт об этом

**Тест:** Заполните заявку как клиент. Убедитесь, что она появилась в CRM с правильными данными.

---

## Проверка 4: Fallback работает
- [ ] Бот понимает, когда не знает ответа
- [ ] Есть чёткий переход на человека («Сейчас подключу специалиста»)
- [ ] Оператор получает контекст (историю диалога)

**Не делайте:** «Я не понял, повторите» в бесконечном цикле.

---

## Проверка 5: Данные клиентов защищены
- [ ] Я знаю, где хранятся данные (РФ / за рубежом)
- [ ] Для медицины/финансов: серверы в РФ, шифрование
- [ ] Есть политика обработки персональных данных

**152-ФЗ:** Для медицинских данных — только on-premise размещение в РФ.

---

## Проверка 6: Команда готова
- [ ] Операторы знают, как работать с ботом
- [ ] Есть инструкция на случай сбоев
- [ ] Назначен ответственный за мониторинг

**Чек-лист для операторов:**
1. Как посмотреть диалоги бота
2. Когда и как перехватывать диалог
3. Куда сообщать об ошибках

---

## Проверка 7: Есть план развития
- [ ] Я знаю, что добавлю через месяц (новые сценарии? интеграция?)
- [ ] Есть метрики для отслеживания (конверсия, время ответа, CSAT)
- [ ] Запланирована еженедельная проверка диалогов

**Правило:** Бот, который не обновляется, устаревает через 3 месяца.

---

## Что делать, если не все пункты отмечены?

**Отмечено 6-7:** Запускайте, но доработайте недостающее в первые 2 недели.

**Отмечено 4-5:** Исправьте критичные пункты (2, 3, 4) перед запуском.

**Отмечено меньше 4:** Отложьте запуск. Провал почти гарантирован.

---

## Бонус: Мини-аудит за 5 минут

Отправьте бота другу/коллеге, который не участвовал в разработке. Попросите:
1. Задать 3 вопроса из реальной практики
2. Оценить скорость ответа (медленно / нормально / быстро)
3. Сказать, доверил бы он боту свою заявку

Если 2 из 3 ответов положительные — вы готовы.
